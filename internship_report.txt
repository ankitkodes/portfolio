CHAPTER 1: INTRODUCTION
The rapid growth of the internet and mobile technology has fundamentally transformed the way
businesses operate and consumers shop. E-commerce, or electronic commerce, has emerged as one of
the most dynamic and influential sectors of the global economy. From small independent retailers to
large multinational corporations, businesses of every scale have embraced online platforms to reach
wider audiences, streamline operations, and enhance customer experience.
This internship report documents the work undertaken during the internship period at Simplilearn
Solutions Pvt. Ltd., where a fully functional e-commerce web application was designed and developed
from scratch. The project was conceived and implemented using a modern full-stack technology
approach combining Node.js, Express.js, MongoDB, React.js, and raw CSS — collectively enabling a
robust, scalable, and user-friendly online shopping platform.
The developed application supports two distinct types of users: administrators and shoppers.
Administrators have the ability to register, log in, and manage their product catalogue by listing new
products with relevant details. Regular users (shoppers) can register, log in, browse the product listing,
add items to their shopping cart, and proceed to checkout. This dual-role architecture closely mirrors the
real-world structure of popular e-commerce platforms.
This report presents a comprehensive account of the internship experience, the project objectives, the
architecture and technology stack employed, the individual features implemented, challenges faced
during development, and the knowledge gained throughout the process.

CHAPTER 3: PROJECT DESCRIPTION
3.1 Project Title
E-Commerce Website — A full-stack web application enabling administrators to list and manage
products and users to browse, add to cart, and purchase products online.

3.2 Problem Statement
Small and medium-sized businesses often lack affordable, customisable e-commerce platforms that give
them full control over their product listings and sales pipeline. Off-the-shelf solutions either impose high
subscription costs or limit customisation. The goal of this project was to build a bespoke,
open-architecture e-commerce platform that any business could own and extend.

3.3 Scope of the Project
The project covers the complete lifecycle of an online shopping experience, including:
• User authentication and authorisation (separate admin and user roles).
• Product catalogue management by administrators.
• Product browsing and search by shoppers.
• Shopping cart management (add, remove, update items).
• Checkout process with order summary.

3.4 Vision & Mission
Vision: To provide an open, scalable, and developer-friendly e-commerce platform that empowers small
businesses to sell online without vendor lock-in.
Mission: To build a clean, maintainable full-stack application that separates admin and shopper concerns
while delivering a seamless purchasing experience.

CHAPTER 4: SYSTEM ARCHITECTURE & TECHNOLOGY STACK
4.1 Architecture Overview
The application follows the client-server architecture with a clear separation between the front-end
(React.js) and the back-end (Node.js + Express.js). The back-end exposes a RESTful API that is
consumed by the React front-end. Data is persisted in a MongoDB database hosted locally (or via
MongoDB Atlas in the cloud environment).
The architecture can be summarised in three tiers:
• Presentation Layer: React.js components styled with raw CSS, running in the browser.
• Application Layer: Node.js and Express.js server handling routing, business logic, authentication
middleware, and API responses.
• Data Layer: MongoDB collections for users, products, carts, and orders.

CHAPTER 5: FEATURES IMPLEMENTED
5.1 Admin Features
The admin side of the application provides the following capabilities:

5.1.1 Admin Signup & Login
Administrators can create a new account by providing their name, email address, and a secure password.
The password is hashed using bcrypt before being stored in the MongoDB database. On subsequent
visits, admins can log in using their registered credentials. Upon successful authentication, a JWT
(JSON Web Token) is issued and stored on the client side to maintain the session.

5.1.2 Product Listing (Add Products)
Once authenticated, an admin can add new products to the platform. Each product entry includes the
following fields:
• Product Name
• Description
• Price
• Stock Quantity
• Product Image URL
• Category
Products added by the admin are immediately visible to all shoppers browsing the platform.

5.2 User (Shopper) Features
The shopper-facing side of the application provides a complete online shopping experience:

5.2.1 User Signup & Login
New shoppers can register on the platform by providing their name, email, and password. Similar to the
admin flow, the password is hashed securely before storage. Returning users can log in to access their
personalised shopping experience. Authentication is managed via JWT tokens.

5.2.2 Browse Products
Authenticated users can view the complete product catalogue on the home page. Each product card
displays the product image, name, price, and a brief description, giving shoppers all the information they
need to make a purchase decision.

5.2.3 Add to Cart
Users can add any product to their shopping cart with a single click. The cart dynamically updates to
reflect the selected items, their quantities, and the running total price. Users can also adjust the quantity
of each item or remove items from the cart before proceeding.

5.2.4 Checkout
From the cart page, users can proceed to checkout. The checkout flow summarises the order — listing
all selected products, quantities, individual prices, and the grand total. The system confirms the order
and presents a success notification to the user upon completion.

CHAPTER 7: IMPLEMENTATION DETAILS
7.1 Project Structure
The project is organised into two main directories — a backend/ folder containing the Node.js/Express.js
server and a frontend/ folder housing the React.js application.
Backend Directory Structure:
• server.js — Entry point; sets up Express app, connects to MongoDB, and starts the HTTP server.
• routes/ — Contains separate route files for auth (authRoutes.js), products (productRoutes.js), cart
(cartRoutes.js), and orders (orderRoutes.js).
• controllers/ — Business logic for each route.
• models/ — Mongoose schema definitions for User, Product, Cart, and Order.
• middleware/ — Authentication middleware using JWT verification.
Frontend Directory Structure:
• src/components/ — Reusable React components such as Navbar, ProductCard, CartItem, etc.
• src/pages/ — Page-level components: Home, Login, Signup, Cart, Checkout, AdminDashboard.
• src/services/ — Axios API service functions for communicating with the back-end.
• src/App.js — Root component with React Router configuration.

7.2 Authentication Flow
Authentication was implemented using JSON Web Tokens (JWT). When a user or admin logs in
successfully, the server signs a JWT containing the user's ID and role. This token is stored in
localStorage on the client side. For every protected API request, the React application attaches the token
in the Authorization header. The server's auth middleware verifies the token before granting access to
the protected route.

7.4 Front-End Implementation
The front-end was built entirely with React.js using functional components and React Hooks (useState,
useEffect, useContext). Routing between pages was handled by react-router-dom. Global state such as
the logged-in user and cart contents was managed using React Context API, avoiding the need for an
external state management library.
All styling was done using raw CSS (no frameworks such as Bootstrap or Tailwind). Custom CSS files
were written for each component, ensuring full control over the visual design. Responsive design
principles were applied to ensure the application renders correctly on both desktop and mobile screen
sizes.

CHAPTER 9: CHALLENGES & LEARNINGS
9.1 Challenges Encountered
9.1.1 Role-Based Access Control
One of the primary challenges was implementing a clean role-based access control (RBAC) system
where admins and users have mutually exclusive capabilities. This required careful design of the JWT
payload (embedding the role field) and writing Express.js middleware that checks the role before
allowing access to admin-only routes.

9.1.2 Synchronising Cart State
Keeping the cart state synchronised between the React front-end and the MongoDB back-end —
especially when a user adds or removes items rapidly — required thoughtful use of useEffect hooks and
async state updates. Race conditions were mitigated by debouncing rapid successive API calls.

9.1.3 Raw CSS Responsiveness
Building a responsive layout using only raw CSS (without a framework) was considerably more
involved than expected. Flexbox and CSS Grid were used extensively to achieve a layout that adapts
gracefully to different screen sizes.
